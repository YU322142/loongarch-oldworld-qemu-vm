# LoongArch 旧世界 QEMU VM 启动方案 20260608

本版本包含可开源的脚本和文档，用于搭建 Windows 主机上的 LoongArch 旧世界 ABI1.0 Linux/X11 可见 QEMU 窗口测试环境。

包含内容：

- 可见 QEMU 窗口启动脚本。
- 默认网络、声音和宿主机共享目录。
- Loongnix Desktop mini qcow2 下载与校验脚本。
- 工作盘生成、快照和重置脚本。
- 完整使用指南，说明如何把待测软件放进虚拟机并执行人工验收。
- 补充脚本作用、QEMU 路径查找规则、`-QemuDir` 手动启动方式和 PowerShell 执行策略说明。
- 调整 `scripts/Install-Qemu-Windows.ps1`：通过 winget 安装 QEMU 后默认复制到本仓库的 `tools/qemu/`，让下载镜像、创建工作盘、启动和重置脚本优先从项目目录找到 QEMU；传入 `-NoCopyToRepo` 可跳过本地复制。`tools/qemu/` 仍被 Git 忽略并排除在 Release 之外。
- 优化 `scripts/Download-LoongnixImage.ps1` 在找不到 `qemu-img.exe` 时的提示：明确说明 Loongnix 镜像已下载并校验成功，只是尚未创建工作盘，并给出安装 QEMU、传入 `-QemuDir` 或使用 `-SkipWorkDisk` 的下一步命令。
- Loongnix mini 首次在 `tty1` 中用 `root` 启用 SSH 的实测命令：`systemctl enable ssh` 与 `systemctl start ssh`。
- 新增 `shared/setup-loongnix-test-desktop.sh` guest 端一键配置脚本，可安装轻量 X11 桌面、启用 SSH、配置中文 locale、中文字体、LightDM、Loongnix X11 Test Desktop、`xfwm4` 合成器、Xfce panel StatusNotifier/systray 托盘、LXTerminal、Xfe 和 `xfce4-notifyd`。
- Loongnix mini 停在 `tty1` 时安装/启用 LightDM + Loongnix X11 Test Desktop 的完整流程。该会话使用 `xfwm4 --compositor=on`，可避免 ClassIsland/Avalonia 透明窗口在无合成器环境下出现黑底；Xfce panel 的 StatusNotifier 插件提供 `org.kde.StatusNotifierWatcher`，用于托盘图标验收。
- 固定 `org.freedesktop.Notifications` 到 `xfce4-notifyd`，避免 KDE Plasma 通知服务在轻量会话中抢占 D-Bus 名称后退出，影响通知和托盘相关测试。
- 补充共享盘在 guest 中常见为 `/dev/vdb1`、推荐挂载到 `/mnt/hostshare`，以及 QEMU `fat:rw` 共享盘运行中新文件不一定实时刷新的说明。
- .NET/Avalonia 应用测试清单。

不包含内容：

- QEMU 二进制和 EDK2 固件。
- Loongnix 系统镜像。
- 虚拟机工作盘。
- 用户测试软件、运行库或日志；`shared/setup-loongnix-test-desktop.sh` 作为本仓库开源脚本会包含在内。

请按 README 安装 QEMU，并从 Loongnix 官方目录下载系统镜像。
