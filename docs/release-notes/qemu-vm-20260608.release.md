# LoongArch old-world QEMU VM launcher 20260608

## 中文

这是用于 LoongArch 旧世界 ABI1.0 Linux/X11 软件测试的 QEMU 可见窗口启动和轻量桌面配置方案。

本 Release 只包含脚本和文档：

- Windows 下启动可见 LoongArch QEMU 窗口。
- 默认启用网络、声音和宿主机共享目录。
- 下载并校验 Loongnix Desktop mini qcow2。
- 生成可写工作盘，支持快照和重置。
- 提供完整使用指南，说明如何把待测软件放进虚拟机并执行人工验收。
- 补充脚本作用、QEMU 路径查找规则、`-QemuDir` 手动启动方式和 PowerShell 执行策略说明。
- `scripts/Download-LoongnixImage.ps1` 改为默认 64 路 HTTP Range 并发下载；分段连接被截断时会从已写入位置继续请求剩余 byte range，可用 `-Connections 1..64` 调整线程数，用 `-DownloadRetries` 调整请求重试次数，服务器不支持 Range 时自动回退单连接。
- 新增 `Start-Loongnix-Desktop.ps1 -NoWait`，用于启动 QEMU 后立即返回 PowerShell；默认等待 QEMU 窗口关闭，并提示串口日志在 GRUB 交给 Linux 后可能停止刷新。
- 补充 QEMU 可见窗口限制：启动过程中不要最大化；桌面运行过程中也建议保持固定窗口大小，避免 SDL 窗口缩放导致鼠标点击位置偏移。
- 调整 `scripts/Install-Qemu-Windows.ps1`：支持 `-QemuDir` 指定已有 QEMU 目录，并默认尝试通过 winget `--location` 安装到本仓库的 `tools/qemu/`；如果 installer 不支持 location，会退回系统安装后复制。传入 `-NoCopyToRepo` 可跳过本地复制。`tools/qemu/` 仍被 Git 忽略并排除在 Release 之外。
- 修复 `scripts/Install-Qemu-Windows.ps1` 把 winget 下载/安装输出误当作退出码的问题，避免出现 `winget install failed with exit code Found QEMU ... Successfully installed 0` 这类误报。
- 优化 `scripts/Download-LoongnixImage.ps1` 在找不到 `qemu-img.exe` 时的提示：明确说明 Loongnix 镜像已下载并校验成功，只是尚未创建工作盘，并给出安装 QEMU、传入 `-QemuDir` 或使用 `-SkipWorkDisk` 的下一步命令。
- 记录 Loongnix mini 首次在 `tty1` 中用 `root` 启用 SSH 的实测命令：`systemctl enable ssh` 与 `systemctl start ssh`。
- 新增 `shared/setup-loongnix-test-desktop.sh` guest 端一键配置脚本，可安装轻量 X11 桌面、启用 SSH、配置中文 locale、中文字体、LightDM、Loongnix X11 Test Desktop、`xfwm4` 合成器、Xfce panel StatusNotifier/systray 托盘、LXTerminal、Xfe 和 `xfce4-notifyd`。
- 新增并提交 `shared/pic.png` 默认测试桌面壁纸；一键配置脚本会复制它并用 `feh` 设置，文档说明壁纸可能需要拖动窗口覆盖桌面区域触发重绘后才显示。
- 壁纸设置改为安装 `loongnix-apply-wallpaper` helper；若壁纸先显示后变为灰色，新版会话会在窗口管理器、面板、终端和文件管理器启动完成后补设壁纸，避免后续 X11 组件覆盖背景。
- 壁纸最终改为默认安装并启动 `xfdesktop4`，通过 Xfce 桌面背景管理器持久管理 `shared/pic.png`；`feh` 仅作为回退方案。
- 补充 Loongnix mini 停在 `tty1` 时安装/启用 LightDM + Loongnix X11 Test Desktop 的完整流程。该会话使用 `xfwm4 --compositor=on`，可避免 ClassIsland/Avalonia 透明窗口在无合成器环境下出现黑底；Xfce panel 的 StatusNotifier 插件提供 `org.kde.StatusNotifierWatcher`，用于托盘图标验收。
- 固定 `org.freedesktop.Notifications` 到 `xfce4-notifyd`，避免 KDE Plasma 通知服务在轻量会话中抢占 D-Bus 名称后退出，影响通知和托盘相关测试。
- 补充共享盘在 guest 中常见为 `/dev/vdb1`、推荐挂载到 `/mnt/hostshare`，以及 QEMU `fat:rw` 共享盘运行中新文件不一定实时刷新的说明。
- 提供 ClassIsland/OpenRemoteShouter 这类 .NET/Avalonia 软件的测试清单。

本 Release 不包含：

- QEMU 二进制和 EDK2 固件。
- Loongnix 系统镜像。
- 虚拟机工作盘。
- 用户测试软件、运行库或日志；`shared/setup-loongnix-test-desktop.sh` 和 `shared/pic.png` 作为本仓库开源辅助内容会包含在内。

请按 README 安装 QEMU，并从 Loongnix 官方目录下载系统镜像。

## English

This Release contains a visible QEMU window launcher and lightweight desktop setup for LoongArch old-world ABI1.0 Linux/X11 application testing.

Included:

- Start a visible LoongArch QEMU window on Windows.
- Enable networking, audio, and a host shared folder by default.
- Download and verify the Loongnix Desktop mini qcow2 image.
- Generate a writable work disk with snapshot and reset support.
- Provide a full usage guide for moving test packages into the VM and performing manual acceptance.
- Add script role reference, QEMU path lookup rules, `-QemuDir` manual startup examples, and PowerShell execution-policy notes.
- `scripts/Download-LoongnixImage.ps1` now uses 64 parallel HTTP Range connections by default; when a segment connection is cut short, it continues from the bytes already written. Adjust with `-Connections 1..64`, tune request retries with `-DownloadRetries`, and fall back automatically to a single connection when the server does not support Range.
- Add `Start-Loongnix-Desktop.ps1 -NoWait` for returning to PowerShell immediately after starting QEMU; the default behavior still waits until the QEMU window closes and now explains that the serial log may stop updating after GRUB hands control to Linux.
- Add visible QEMU window limits: do not maximize during boot, and keep a fixed window size during desktop testing to avoid SDL resize-related mouse click offset.
- Update `scripts/Install-Qemu-Windows.ps1`: it now supports `-QemuDir` for an existing QEMU directory and tries to install QEMU into this repository's `tools/qemu/` with winget `--location` by default. If the installer does not support location, it falls back to a system install and copies QEMU afterward. Pass `-NoCopyToRepo` to skip the local copy. `tools/qemu/` remains ignored by Git and excluded from Release packages.
- Fix `scripts/Install-Qemu-Windows.ps1` treating winget download/install output as the exit code, avoiding false failures such as `winget install failed with exit code Found QEMU ... Successfully installed 0`.
- Improve `scripts/Download-LoongnixImage.ps1` guidance when `qemu-img.exe` is missing: it now makes clear that the Loongnix image has been downloaded and verified, only the work disk is missing, and shows the next commands for installing QEMU, passing `-QemuDir`, or using `-SkipWorkDisk`.
- Document the tested Loongnix mini `tty1` SSH enablement commands run as `root`: `systemctl enable ssh` and `systemctl start ssh`.
- Add the guest-side `shared/setup-loongnix-test-desktop.sh` one-shot setup script for installing the lightweight X11 desktop, enabling SSH, and configuring Chinese locale/fonts, LightDM, Loongnix X11 Test Desktop, the `xfwm4` compositor, Xfce panel StatusNotifier/systray support, LXTerminal, Xfe, and `xfce4-notifyd`.
- Add and track `shared/pic.png` as the default test desktop wallpaper; the one-shot setup script copies it and applies it with `feh`, and the docs explain that the wallpaper may need a window dragged across the desktop area to trigger repainting.
- Wallpaper setup now installs a `loongnix-apply-wallpaper` helper; if the wallpaper appears first and then turns gray, the updated session reapplies it after the window manager, panel, terminal, and file manager finish starting so later X11 components do not overwrite the background.
- Wallpaper handling now installs and starts `xfdesktop4` by default so Xfce's desktop background manager persistently owns `shared/pic.png`; `feh` is kept only as a fallback.
- Add the full LightDM + Loongnix X11 Test Desktop installation/enabling workflow for Loongnix mini when it stops at `tty1`. The session uses `xfwm4 --compositor=on` to avoid black backgrounds on ClassIsland/Avalonia transparent windows, and the Xfce panel StatusNotifier plugin provides `org.kde.StatusNotifierWatcher` for tray-icon acceptance.
- Pin `org.freedesktop.Notifications` to `xfce4-notifyd` so KDE Plasma's notification service does not claim the D-Bus name and exit inside the lightweight session, which can affect notification and tray-related tests.
- Document that the host shared disk commonly appears as `/dev/vdb1`, should be mounted at `/mnt/hostshare`, and QEMU `fat:rw` may not show host-side new files immediately while the VM is running.
- Provide a test checklist for .NET/Avalonia applications such as ClassIsland and OpenRemoteShouter.

Not included:

- QEMU binaries or EDK2 firmware.
- Loongnix system images.
- Writable VM work disks.
- User test software, runtimes, or logs; `shared/setup-loongnix-test-desktop.sh` and `shared/pic.png` are included as open-source repository helpers.

Install QEMU and download the system image from the official Loongnix directory as described in README.
