# LoongArch Old-World QEMU VM Launcher 20260608

This version contains open-source scripts and documentation for setting up a visible Windows-hosted LoongArch old-world ABI1.0 Linux/X11 QEMU-window test environment.

Included:

- Visible QEMU window launcher script.
- Default networking, audio, and host shared folder.
- Loongnix Desktop mini qcow2 download and checksum script.
- Work-disk creation, snapshot, and reset scripts.
- Full usage guide for moving test packages into the VM and performing manual acceptance.
- Script role reference, QEMU path lookup rules, `-QemuDir` manual startup examples, and PowerShell execution-policy notes.
- New `Start-Loongnix-Desktop.ps1 -NoWait` option for returning to PowerShell immediately after starting QEMU; the default behavior still waits until the QEMU window closes and now explains that the serial log may stop updating after GRUB hands control to Linux.
- Added visible QEMU window limits: do not maximize during boot, and keep a fixed window size during desktop testing to avoid SDL resize-related mouse click offset.
- Updated `scripts/Install-Qemu-Windows.ps1`: it now supports `-QemuDir` for an existing QEMU directory and tries to install QEMU into this repository's `tools/qemu/` with winget `--location` by default. If the installer does not support location, it falls back to a system install and copies QEMU afterward. Pass `-NoCopyToRepo` to skip the local copy. `tools/qemu/` remains ignored by Git and excluded from Release packages.
- Fixed `scripts/Install-Qemu-Windows.ps1` treating winget download/install output as the exit code, avoiding false failures such as `winget install failed with exit code Found QEMU ... Successfully installed 0`.
- Improved `scripts/Download-LoongnixImage.ps1` guidance when `qemu-img.exe` is missing: it now makes clear that the Loongnix image has been downloaded and verified, only the work disk is missing, and shows the next commands for installing QEMU, passing `-QemuDir`, or using `-SkipWorkDisk`.
- Tested Loongnix mini `tty1` SSH enablement commands run as `root`: `systemctl enable ssh` and `systemctl start ssh`.
- New guest-side `shared/setup-loongnix-test-desktop.sh` one-shot setup script for installing the lightweight X11 desktop, enabling SSH, and configuring Chinese locale/fonts, LightDM, Loongnix X11 Test Desktop, the `xfwm4` compositor, Xfce panel StatusNotifier/systray support, LXTerminal, Xfe, and `xfce4-notifyd`.
- Added and tracked `shared/pic.png` as the default test desktop wallpaper; the one-shot setup script copies it and applies it with `feh`, and the docs explain that the wallpaper may need a window dragged across the desktop area to trigger repainting.
- Full LightDM + Loongnix X11 Test Desktop installation/enabling workflow when Loongnix mini stops at `tty1`. The session uses `xfwm4 --compositor=on` to avoid black backgrounds on ClassIsland/Avalonia transparent windows, and the Xfce panel StatusNotifier plugin provides `org.kde.StatusNotifierWatcher` for tray-icon acceptance.
- Pin `org.freedesktop.Notifications` to `xfce4-notifyd` so KDE Plasma's notification service does not claim the D-Bus name and exit inside the lightweight session, which can affect notification and tray-related tests.
- Notes that the host shared disk commonly appears as `/dev/vdb1`, should be mounted at `/mnt/hostshare`, and QEMU `fat:rw` may not show host-side new files immediately while the VM is running.
- .NET/Avalonia application test checklist.

Not included:

- QEMU binaries or EDK2 firmware.
- Loongnix system image.
- Writable VM work disk.
- User test software, runtimes, or logs; `shared/setup-loongnix-test-desktop.sh` and `shared/pic.png` are included as open-source repository helpers.

Install QEMU and download the system image from the official Loongnix directory as described in README.
